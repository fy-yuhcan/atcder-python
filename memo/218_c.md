# 問題番号(カテゴリ)
218C - Shapes　回転の座標変換

# 方針
- グリッドの90度回転は「縦横が入れ替わり、片方が逆順になる」と考える
- new_i = j
- new_j は「i を逆順にしたい」→ 0..N-1 を N-1..0 にする → N-1-i
- 四隅や数点でチェックして確信を持つ

## カテゴリ
- 座標変換
- 実装テク（回転・反転の一般パターン）

## 詰まったところ
- なぜ N-1-i になるか直感的に結びつかなかった
- 「縦横入れ替え」と「逆順」の2段階で分けて考える発想がなかった
- #の記号の座標のみを保存する発想
- 左上で固定する、正気化

## 次に出たら
- まず小さい N でマスに (i,j) を書いて回転後の位置を手で確認する
- 「縦横が入れ替わる」→ `new_i` or `new_j` のどちらかを旧 `j` にする
- もう一方は「逆順かどうか」だけを考え、0..N-1 → N-1..0 の式を作る
- 四隅や数点で検算してからコードにする
- 正規化
```python
# 正規化：左上を(0,0)に持ってくる
def normalize(cells):
    min_i = min(i for i, j in cells)
    min_j = min(j for i, j in cells)
    normalized = sorted((i - min_i, j - min_j) for i, j in cells)
    return normalized
```
- 90度のgrid回転
```python
# 90度回転(N-1-iはjの変換)
def rotate(cells):
    return [(j, N - 1 - i) for i, j in cells]
```
