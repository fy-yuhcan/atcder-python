# 問題番号(カテゴリ)
ABC216 C - Many Balls（二進法・逆操作）
# 方針
0からNでなくNから0になるように操作を繰り返す
ループしながらNが0になるまで
Nが奇数の時+1
Nが偶数の時*2と考えられる

A の回数 = N の 2 進数の 1 の個数
B の回数 ≒ N のビット数 - 1
## カテゴリ
- 逆方向の貪欲（逆操作）
-  2進法 / ビット
- 数学（logN のイメージ）
- シミュレーション

## 詰まったところ
最初は 前向き（0 → N）に貪欲 しようとした
「2倍できるだけして、超えそうになったら +1 で埋める」みたいな方針
これだと N が大きいと、最後の +1 が大量に必要になって 120 回以内に収まらない
