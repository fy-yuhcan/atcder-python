# ABC213 C - Reorder Cards（座標圧縮）

# 方針
- 二次元配列を作らない(h,wの量が大きいため)
- 数字がある行列のみをソート、何番目かを見るだけ,行列削除をシミュレートしない
- 行列それぞれで座標圧縮

## カテゴリ
- 座標圧縮
- 二分探索/dict
- ソート、セット

## 詰まったところ
- h,wの二次元配列を作って削除シミュレートしようとした、制約的に不可能(10*9*10*9)
- 行列を削除する操作を
->最終的な行番号 = 数字の中だけの順位
と理解できなかった
- 座標圧縮が曖昧
    - set / bisect_left / dict の役割が曖昧だった
    - set(x)：重複を消して「使う行だけ」の集合にする
    - sorted(set(x))：小さい順に並べて順位をつけられる状態にする
    - bisect_left / row_id：
    - 「この行が何番目に小さいか？」＝圧縮後の番号

## 次に出たら
制約が異常に大きい二次元配列は作らない
日本語的に
    - 「行・列を削除する」
    - 「詰める」
    - 「欠番をなくす」
みたいなワードが出たら
→ 「座標圧縮で“使う座標だけ1,2,3,…に振り直せないか？」 を疑う
- bisectで座標圧縮
```
import bisect

for i in range(N):
    cx = bisect.bisect_left(rows, X[i]) + 1
    cy = bisect.bisect_left(cols, Y[i]) + 1
    print(cx, cy)
```
